# n8n Docker Setup with ngrok

This project helps you run **n8n using Docker** and expose it to the internet using **ngrok**.

n8n is an open-source workflow automation tool used to automate APIs, integrations, and tasks.

ngrok allows you to securely expose your **local machine to the internet** using a public HTTPS URL.

# Prerequisites

Before running this project install the following tools.

---

## 1 Install Docker Desktop

Download Docker Desktop:

https://www.docker.com/products/docker-desktop/

Verify installation:

```bash
docker --version
```

---

## 2 Install ngrok

Download ngrok:

https://ngrok.com/download

Verify installation:

```bash
ngrok --version
```

---

## 3 Create ngrok Account

Create a free account:

```
https://dashboard.ngrok.com
```

After login copy your **Auth Token**.

Run the following command to authenticate ngrok:

```bash
ngrok config add-authtoken YOUR_AUTH_TOKEN
```

# Environment Variables

Inside the project you will find a file:

```
.env
```

Example configuration:

```
DOMAIN_NAME=your-ngrok-domain.ngrok-free.dev
GENERIC_TIMEZONE=Europe/Berlin
SSL_EMAIL=user@example.com
```

⚠️ Important

Do **NOT use the example domain above**.

Each user must configure **their own ngrok domain**.

Example:

```
DOMAIN_NAME=abcd1234.ngrok-free.dev
```

---

# Environment Variable Explanation

## DOMAIN_NAME

This is the **public domain generated by ngrok**.

Example ngrok output:

```
Forwarding https://abcd1234.ngrok-free.dev -> http://localhost:5678
```

Set the domain like this:

```
DOMAIN_NAME=abcd1234.ngrok-free.dev
```

---

## GENERIC_TIMEZONE

Defines the timezone used by n8n workflows.

Example:

```
GENERIC_TIMEZONE=Asia/Kolkata
```

Other examples:

```
GENERIC_TIMEZONE=Europe/Berlin
GENERIC_TIMEZONE=America/New_York
```

---

## SSL_EMAIL

Email used for SSL related configuration.

Example:

```
SSL_EMAIL=your_email@gmail.com
```

# Running the Project

## Step 1 Start Docker

Run the following command:

```bash
docker compose up -d
```

This starts the n8n container in the background.

---

## Step 2 Open n8n

Open the browser and go to:

```
http://localhost:5678
```

n8n editor will open.

By default **n8n runs on port 5678**.  

---

## Step 3 Start ngrok

Expose your local n8n instance to the internet.

```bash
ngrok http 5678
```

Example output:

```
Forwarding https://abcd1234.ngrok-free.dev -> http://localhost:5678
```

Now your n8n instance is accessible publicly.

```

   ↓
ngrok public URL
   ↓
localhost:5678
   ↓
Docker container
   ↓
n8n workflow engine
```

# Common Issues

## ngrok command not found

Ensure ngrok is installed and added to PATH.

---

## Docker not running

Start Docker Desktop and try again.

---

## Port already in use

Another application may already be using port 5678.

Stop the application or change the port in `docker-compose.yml`.
